Abstract

The proliferation of social media platforms has fundamentally transformed the way individuals express their opinions, sentiments, and attitudes toward various topics, entities, brands, and sociopolitical events. Among these platforms, Reddit stands out as a massive, decentralized aggregator of diverse, community-driven discussions. Extracting meaningful, quantitative sentiment from such vast arrays of unstructured text data presents significant opportunities for market research, public relations, financial forecasting, and sociological trend analysis.

This research paper presents REDSEA (Reddit Sentiment Analysis), an optimized, high-throughput, real-time web application engineered to fetch, process, and visualize sentiment trends from the Reddit platform. Developed utilizing a modern technology stack comprising Python, the Flask web framework, PRAW (Python Reddit API Wrapper), and the Natural Language Toolkit (NLTK) VADER lexicon, REDSEA efficiently aggregates data and computes sentiment scores for both overarching post titles and highly granular nested comments.

To systematically address the profound computational bottlenecks associated with large-scale text analysis and external network latency, the system architecture employs advanced concurrent processing via ThreadPoolExecutor paradigms and implements multi-layered, aggressive caching mechanisms. The empirical results demonstrate that REDSEA can analyze complex time-series sentiment with exceptionally high throughput and minimal latency. This paper comprehensively details the theoretical foundations of sentiment analysis, the architectural choices of the REDSEA system, mathematical justifications for lexical scoring, implementation optimizations, and rigorous empirical evaluations.

The system uses a thread pool to analyze post chunks in parallel. This is useful because Reddit comment access can involve waiting on external network responses. While Python's Global Interpreter Lock limits true CPU-bound parallelism, I/O-bound tasks can still benefit from multiple worker threads because one thread can wait while another continues progress.

Chapter 4: System Implementation (Extended Architecture)

4.16 End-to-End Data Flow Explanation

The REDSEA application can be understood as an end-to-end data flow beginning with a user query and ending with a rendered analytical dashboard. When the user enters a company name, the browser sends a POST request to the Flask server. The server does not immediately produce a sentiment value; instead, it first interprets the query as a retrieval instruction. This separation between retrieval and analysis is important because sentiment analysis is only meaningful when the input corpus is relevant to the user's topic.

The backend then uses PRAW to communicate with Reddit. PRAW returns submission objects containing post titles, links, timestamps, and comment structures. REDSEA converts those external API objects into local Python dictionaries. This conversion is a useful design step because it prevents the presentation layer from depending directly on Reddit's internal object structure. Once data has been normalized into dictionaries, it can be sorted, scored, aggregated, and rendered more predictably.

After title and comment sentiment values are computed, REDSEA calculates two categories of output. The first is post-level output, where each post has a title score and an average comment score. The second is time-series output, where posts are grouped into chronological intervals and counted as positive, negative, or neutral. The first output supports detailed inspection, while the second output supports quick pattern recognition.

User Query -> Reddit Search -> Post Objects -> Normalized Dictionaries -> Sentiment Scoring -> Time Bucketing -> HTML + Chart Output

Chapter 4: System Implementation (Algorithmic View)

4.18 Core Algorithm of REDSEA

The REDSEA workflow may be represented algorithmically. This makes the design easier to understand for academic review because it separates logical operations from programming-language syntax. The algorithm begins with validation, moves into retrieval, performs sentiment scoring, aggregates the output, and returns both detailed and summarized results.

Algorithm: REDSEA Sentiment Analysis
Input: company_name
Output: analyzed_posts, sentiment_time_series

1. If company_name is empty, return empty output.
2. Retrieve recent Reddit posts matching company_name.
3. For each post:
   a. Extract title, URL, created_utc, and top-level comments.
   b. Compute title compound sentiment using VADER.
   c. Compute compound sentiment for selected comments.
   d. Calculate average comment sentiment.
   e. Store normalized post result.
4. Sort analyzed posts by created_utc in descending order.
5. Divide recent time window into fixed intervals.
6. Count positive, negative, and neutral posts in each interval.
7. Render results in HTML and initialize Chart.js data.

The algorithm is intentionally deterministic once Reddit has returned the post set. If the same set of posts and comments is provided, the same VADER scores and chart intervals will be produced. This deterministic behavior is useful for reproducibility, testing, and classroom explanation. The only non-deterministic element is the changing public data source, since Reddit content evolves continuously.

Chapter 4: System Implementation (Complexity and Performance)

4.19 Computational Complexity

The computational cost of REDSEA depends on the number of posts retrieved and the number of comments analyzed per post. If the system retrieves n posts and analyzes c comments for each post, the approximate scoring workload is O(n * (1 + c)), where the extra one represents the title. Since the current system analyzes a fixed number of comments per post, the practical complexity grows linearly with the number of posts.

This linear behavior is important because it makes the system manageable. A project that recursively analyzed entire comment trees could become expensive quickly, because popular Reddit posts may contain hundreds or thousands of comments. REDSEA avoids that by focusing on a bounded subset of top-level comments. This is a deliberate engineering trade-off: the system gives up some depth in exchange for interactive response time.

Chapter 4: System Implementation (Security and Reliability)

4.21 Configuration Safety

REDSEA requires Reddit API credentials. These credentials should not be hardcoded in a production repository. The project uses environment variables loaded through python-dotenv, which is a safer and more flexible approach. Local developers can place credentials in a private .env file, while deployment platforms can inject credentials through environment settings.

Security in this project is not limited to secret handling. User input must also be treated carefully. The company name entered by the user is passed into a search query and rendered back into the template. Flask's template engine provides automatic escaping, which helps reduce injection risk in the browser. However, future versions should also add input length limits, request throttling, and clearer error messages for failed API calls.

Reliability also depends on graceful degradation. If Reddit is unavailable, the application should not crash. If no posts are returned, the user should receive a no-results state. If chart data is empty, the frontend should avoid breaking. These design expectations are part of making REDSEA a dependable academic prototype.

Chapter 5: Results and Analysis (Expanded Evaluation)

5.15 Interpretation of Sentiment Scores

The VADER compound score is a continuous value between -1 and +1. In REDSEA, values above 0.05 are treated as positive, values below -0.05 are treated as negative, and values between those thresholds are treated as neutral. This thresholding strategy is useful because it prevents very small fluctuations from being interpreted as meaningful polarity.

For example, a score of 0.02 may technically be slightly positive, but classifying it as positive would overstate the certainty of the result. Similarly, a score of -0.03 should not be treated as a strong negative signal. The neutral interval therefore acts as a buffer zone. This is important in social media text, where short posts may contain limited emotional evidence.

Score RangeClassificationInterpretation
0.05 to 1.00PositiveText contains favorable or approving sentiment
-0.05 to 0.05NeutralText is factual, mixed, weak, or unclear
-1.00 to -0.05NegativeText contains criticism, dissatisfaction, or unfavorable tone

These classifications should be treated as analytical aids rather than absolute truth. The user must still inspect the original posts when accuracy matters. REDSEA supports this by linking each displayed post to its source.

5.18 Suggested Human Validation Method

Because REDSEA uses a rule-based sentiment analyzer, human validation is important for future research. A suggested validation protocol would begin by selecting a balanced sample of analyzed posts. The sample should include positive, negative, and neutral outputs from different company queries. Each item should then be independently labeled by two or more human reviewers.

The reviewers should label each text as positive, negative, neutral, or unclear. They should also mark whether the text appears sarcastic, ambiguous, off-topic, or context-dependent. This additional tagging would help identify why the automated model succeeds or fails. For example, if many false positives are sarcastic, then future work should focus on sarcasm detection rather than general sentiment scoring.

Validation Steps:
1. Collect sample REDSEA outputs.
2. Remove usernames and unnecessary metadata.
3. Ask reviewers to label sentiment independently.
4. Compare VADER label with human label.
5. Record disagreement causes.
6. Calculate accuracy, precision, recall, and F1-score.

Chapter 5: Results and Analysis (Observed Weaknesses)

5.20 Weaknesses and Practical Constraints

The first weakness is dependence on Reddit API behavior. If credentials are missing, if the API changes, or if rate limits are reached, live retrieval can fail. The second weakness is relevance ambiguity. Searching across all subreddits may return posts that mention the company name without actually discussing the company as a brand or business entity.

The third weakness is model limitation. VADER is strong for many forms of social media text, but it does not deeply understand context, sarcasm, long arguments, or domain-specific references. A fourth weakness is lack of persistent storage. Without a database, REDSEA cannot provide long-term trend history. Each analysis is based on recent retrieval rather than accumulated historical evidence.

These weaknesses are not hidden problems; they are important research findings. They show where the prototype is complete and where future work should focus. A good research paper should be honest about such boundaries because they make the recommendations more credible.

Chapter 5: Results and Analysis (Research Interpretation)

5.21 What the Results Mean

The results produced by REDSEA should be interpreted as signals, not final judgments. A positive chart pattern suggests that recent visible discussion contains more favorable language, but it does not prove that the entire public has a positive opinion. The value of the system is therefore in reducing the time needed to identify possible patterns. Instead of reading dozens of posts manually before forming any impression, a user can first view a chart and a sorted list. The user can then inspect the linked posts to confirm or reject the automated impression. This is a practical human-in-the-loop workflow.

From a research perspective, REDSEA demonstrates that even a lightweight NLP model can become useful when placed inside a well-designed retrieval and visualization pipeline. The pipeline is as important as the model because it determines whether the analysis is timely, accessible, and understandable.
